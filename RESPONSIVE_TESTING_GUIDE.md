## 🔍 測試方法

### 方法 1：使用瀏覽器開發者工具

#### Chrome / Edge
1. 開啟 `homepage-complete-preview.html`
2. 按 `F12` 或 `Cmd+Option+I` (Mac) 開啟開發者工具
3. 點擊「Toggle device toolbar」圖示（📱）或按 `Cmd+Shift+M` (Mac)
4. 選擇預設裝置或自定義尺寸

#### Safari
1. 開啟 Safari > 偏好設定 > 進階 > 勾選「在選單列中顯示開發選單」
2. 開啟 `homepage-complete-preview.html`
3. 開發 > 進入響應式設計模式（`Cmd+Option+R`）
4. 選擇不同的裝置尺寸測試

#### Firefox
1. 開啟 `homepage-complete-preview.html`
2. 按 `F12` 開啟開發者工具
3. 點擊「響應式設計模式」圖示（📱）或按 `Cmd+Option+M` (Mac)
4. 測試不同尺寸

### 方法 2：調整瀏覽器視窗大小
1. 開啟 `homepage-complete-preview.html`
2. 手動拖曳視窗邊緣調整寬度
3. 觀察在不同寬度下的變化：
   - 1200px 以上：桌面版
   - 768px - 1024px：平板版
   - 768px 以下：手機版

### 方法 3：實際裝置測試
1. 將檔案上傳到網頁伺服器
2. 使用手機、平板實際訪問
3. 測試觸控互動和滾動效果

## 🐛 常見問題排查

### 問題 1：手機版導航選單沒有垂直堆疊
**檢查**：
```css
@media (max-width: 768px) {
  .main-navigation ul {
    flex-direction: column;
    gap: var(--spacing-sm);
  }
}
```

### 問題 2：卡片在平板上沒有自動換行
**檢查**：
```css
grid-template-columns: repeat(auto-fit, minmax(250px, 1fr));
```
確保 `minmax` 的最小值適當。

### 問題 3：文字在手機上太小
**檢查**：
- viewport meta tag 是否正確：`<meta name="viewport" content="width=device-width, initial-scale=1.0">`
- 基礎字體大小是否為 16px

### 問題 4：按鈕在手機上太小難以點擊
**解決**：增加 padding 或 min-height
```css
.button {
  min-height: 44px;
  padding: 12px 24px;
}
```
